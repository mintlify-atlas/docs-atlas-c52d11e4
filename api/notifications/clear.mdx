---
title: "notification.clear"
description: "Remove all notifications from the panel and history"
---

The `notification.clear` method removes all notifications from cmux, clearing both the notification panel and notification history.

## Method

```json
{
  "id": "clear-1",
  "method": "notification.clear",
  "params": {}
}
```

## Parameters

This method takes no parameters.

## Response

Returns an empty success result:

```json
{
  "ok": true,
  "result": {}
}
```

## Behavior

Clearing notifications:

### What Gets Cleared

- **All notifications** - Both read and unread
- **Panel display** - Notification panel becomes empty
- **System notifications** - Pending and delivered macOS notifications removed
- **Dock badge** - Unread count badge removed from app icon
- **Workspace badges** - Per-workspace notification indicators cleared

### What Persists

- **Workspace state** - Workspaces, surfaces, and content unchanged
- **Settings** - Notification preferences remain
- **Panel visibility** - Notification panel stays open/closed as before

### Atomicity

- **Single operation** - All notifications cleared in one atomic action
- **No partial clears** - Cannot clear individual notifications via API (UI only)
- **Cannot undo** - Cleared notifications are permanently removed
- **Immediate effect** - UI updates instantly

## Notification Panel Behavior

After clearing:

1. **Panel shows empty state** - "No notifications" message displayed
2. **Badge removed** - Dock icon badge disappears
3. **Workspace indicators cleared** - Unread counts reset to zero
4. **System notification center** - Delivered notifications removed from macOS notification center

```javascript

const listResult = await sendSocketRequest({
  id: "req-2a",
  method: "notification.list",
  params: {}
});

if (count > 0) {
  const confirmed = confirm(`Clear ${count} notification(s)?`);
  
  if (confirmed) {
    await sendSocketRequest({
      id: "req-2b",
      method: "notification.clear",
      params: {}
    });
    console.log(`Cleared ${count} notifications`);
  }

```

### Periodic Cleanup

```javascript
// Clear notifications older than 24 hours
const listResult = await sendSocketRequest({
  id: "req-4a",
  method: "notification.list",
  params: {}
});

const now = Date.now();
const dayAgo = now - (24 * 60 * 60 * 1000);

// Note: createdAt not in v2 response yet, would need to track separately
const oldNotifs = listResult.result.notifications.filter(
  n => n.createdAt && new Date(n.createdAt).getTime() < dayAgo
);

if (oldNotifs.length > 0) {
  await sendSocketRequest({
    id: "req-4b",
    method: "notification.clear",
    params: {}
  });
  console.log(`Cleared ${oldNotifs.length} old notifications`);
}
```

## Python Example

```python
import json
import socket

def clear_notifications(sock_path="/tmp/cmux.sock"):
    """Clear all cmux notifications via socket API."""
    request = {
        "id": "py-clear-1",
        "method": "notification.clear",
        "params": {}
    }
    
    sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
    try:
        sock.connect(sock_path)
        sock.sendall(json.dumps(request).encode() + b"\n")
        
        response = b""
        while b"\n" not in response:
            response += sock.recv(4096)
        
        result = json.loads(response.decode())
        return result.get("ok", False)
    finally:
        sock.close()

# Usage
if clear_notifications():
    print("Notifications cleared successfully")
else:
    print("Failed to clear notifications")
```

func clearAll() {
    guard !notifications.isEmpty else { return }
    let ids = notifications.map { $0.id.uuidString }
    notifications.removeAll()
    center.removeDeliveredNotifications(withIdentifiers: ids)

## Best Practices

1. **Archive before clearing** - Save notification history if needed for audit/debugging
2. **Confirm in interactive contexts** - Prompt user before clearing in UI applications
3. **Use for test isolation** - Clear before each test for predictable state
4. **Schedule periodic clears** - Prevent unbounded growth in long-running sessions
5. **Log clearing events** - Track when and why notifications are cleared

## Limitations

- **No selective clearing** - Cannot clear individual notifications via API
- **No undo** - Cleared notifications cannot be recovered
- **No workspace filtering** - Cannot clear notifications for specific workspace only
- **No timestamp filtering** - Cannot clear by date range

**Note**: Individual notification management (delete single, mark as read, etc.) is only available through the UI currently. Future API methods may support these operations.

## Performance

- **Fast operation** - Typically completes in less than 10ms
- **Scales linearly** - Performance degrades with notification count
- **Main thread execution** - Runs on UI thread but minimal blocking
- **Atomic operation** - All-or-nothing clearing
